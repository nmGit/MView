mView.MDevice module¶

class mView.MDevice.MDevice(name, *args, **kwargs)[source]¶

Bases: PyQt4.QtCore.QThread

MView uses the MDevice class to give all sources of data a common interface with which to interact in the context of MView. These sources of data can be anything including but not limited to LabRad servers, RS232 devices, GPIB Devices, they can even represent the contents of .hdf5 files. Devices in MView are created by instantiating their device drivers. For example, if there are two RS232 devices, we create two instances of the RS232 device driver. This means that only one generic device driver needs to be created for one interface (RS232, LabRad Servers, HDF5 files, etc.) and it can then be applied to all devices that use the same interface.

onAddParameter(*args, **kwargs)[source]¶

Called when when a new parameter is added. It is passed whatever MDevice.addParameter() is passed. (Note: MDevice.onAddParameter() and MDevice.addParameter() are different). This function must return a tuple in the form ((str) Parameter Name, (int)Precision, (str) units)

onBegin()[source]¶

Called at the end of MDevice.begin(). This is called before MView starts. This allows us to configure settings that MView might use while starting. This might include datalog locations or device-specific information.

onLoad()[source]¶

Called at the end of MGui.startGui(), when the main MView GUI has finished loading. This allows the MDevice to configure pieces of MView only available once the program has fully loaded.

prompt(button)[source]¶

Called when a device’s button is pushed. Button is an array which is associated with the button. The array is constructed in the device driver code, and the PyQT button is then appended to the end by MView. The array associated with the button is passed to prompt() in the device driver. The device driver then determines what to do based on the button pushed.
